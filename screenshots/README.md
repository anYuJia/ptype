# 📸 截图说明

## 如何添加截图

将你的截图放在这个文件夹中，并使用以下文件名：

### 必需的截图

1. **demo.png** （推荐尺寸：1200x600）
   - 应用的主界面或最吸引人的功能展示
   - 这张图会显示在 README 的顶部
   - 建议展示：实时打字界面 + 统计面板

2. **english-mode.png** （推荐尺寸：800x400）
   - 英文模式的打字界面
   - 展示：输入框、当前文本、实时统计

3. **chinese-mode.png** （推荐尺寸：800x400）
   - 中文模式的打字界面
   - 可以展示现代文或古文模式

4. **code-mode.png** （推荐尺寸：800x400）
   - 程序员模式的打字界面
   - 展示：代码高亮、Tab/Enter 特殊字符显示
   - 建议选择 Python 或 JavaScript 示例

5. **results.png** （推荐尺寸：800x400）
   - 完成后的统计面板
   - 展示：WPM/CPM 图表、准确率、详细统计

### 截图技巧

- ✅ 使用高分辨率（至少 2x）
- ✅ 确保界面清晰可读
- ✅ 选择有代表性的内容
- ✅ 保持一致的主题/配色
- ✅ 可以使用设计工具添加阴影或边框

### 推荐工具

- **macOS**: Cmd + Shift + 4 (区域截图)
- **Windows**: Win + Shift + S
- **Linux**: Flameshot, Spectacle
- **浏览器**: 开发者工具的设备模拟

### 示例文件结构

```
screenshots/
├── demo.png              # 主演示图 ✨
├── english-mode.png      # 英文模式
├── chinese-mode.png      # 中文模式
├── code-mode.png         # 代码模式
├── results.png           # 统计结果
└── README.md            # 本说明文件
```

## 占位图说明

如果你暂时没有截图，README 中的图片链接会显示占位符。

添加截图后，图片会自动显示在 GitHub 上。

## 更新 README

如果你想使用不同的文件名或添加更多截图：

1. 将截图放在这个文件夹
2. 编辑 `/README.md` 中的图片路径
3. 格式：`![描述](./screenshots/你的文件名.png)`
